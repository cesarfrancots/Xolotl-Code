<objective>
Wire the extended Rust supervisor (plan 05-01) and the notification plugin (plan 05-02) into
the Tauri IPC surface. The `spawn_agent` Tauri command becomes the entry point for the spawn
dialog: it takes task/model/budget, slugifies the task into a branch, calls the new
`spawn_agent_with_config`, and starts the budget-aware event relay. The relay accumulates
cost on every `TurnCompleted` event and injects synthetic `StateChanged(Failed)` + `Error`
events when the budget is exceeded (D-10).

(budget halt) requires the relay to enforce the limit before more turns happen. The
bindings.ts update gives the frontend type-safety on the new signature.

pub(crate) fn spawn_event_relay(
    app_handle: AppHandle,
    agent_id: AgentId,
    handle: AgentHandle,
) {
    let mut rx = handle.subscribe();
    let channel = format!("agent-event:{}", agent_id.0);
    tokio::spawn(async move {
        loop {
            match rx.recv().await {
                Ok(event) => { let _ = app_handle.emit(&channel, &event); }
                Err(RecvError::Lagged(n)) => { /* synthetic events-lost event */ }
                Err(RecvError::Closed) => break,
            }
        }
    });
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend invoke → Rust command | The new spawn_agent takes user-controlled task string and a numeric budget. Both reach `slugify_task` and `spawn_agent_with_config` respectively. |
| TurnCompleted event → cost accumulation | Untrusted upstream events drive the budget enforcement loop. A maliciously crafted high-usage event could blow through the budget on a single turn — accepted (it would still trigger Failed). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 | Tampering | spawn_agent budget_dollars param | mitigate | Validation lives in `spawn_agent_with_config` (plan 05-01 Task 2). Invalid budgets return `SupervisorError::InvalidBudget` which `map_err(\|e\| e.to_string())?` surfaces to the frontend as a typed error. |
| T-5-02 | Tampering | task → branch name (git argument) | mitigate | `slugify_task(&task)` runs before any git call; only `[a-z0-9-]` survives, capped at 40 chars + `agent/` prefix. |
| T-5-06 | Elevation of Privilege | spawn_event_relay handle ownership | mitigate | Handle is cloned into the spawned tokio task; `event_tx.try_send` is non-blocking — a poisoned cumulative_cost mutex would panic only inside that task, contained from the WebView. Acceptable given Mutex<f64> has no panicking accessors apart from `.lock()`. |
