## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend → Tauri plugin permissions | Capabilities file gates which Tauri APIs JS can call. Granting `notification:default` exposes the OS toast API to the WebView. |
| Filesystem access | `fs:default` previously had no path scope — any reachable path was implicitly allowed for the configured fs operations. CR-04 narrows to AppData/AppConfig/$HOME/.xolotl-code. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-03 | Spoofing | Notification content (frontend hook will inject title) | accept | Truncation enforced in JS hook layer (plan 05-06). WinRT toasts render plain text — no HTML execution surface. |
| T-5-04 | Information Disclosure | fs:default without path scope (Phase 4 carryover, CR-04) | mitigate | Task 1 step 4 replaces bare `fs:default` with a path-scoped allow list covering only AppData, AppConfig, and `$HOME/.xolotl-code`. Out-of-scope reads/writes are denied by Tauri ACL. |
| T-5-05 | Denial of Service | Poisoned mutex panic crashing the Tauri process (CR-01) | mitigate | Task 2 replaces `.lock().unwrap()` calls in permission_prompter.rs with `.lock().map_err(...)?` so a poisoned mutex returns an error instead of aborting the process. |
</threat_model>
