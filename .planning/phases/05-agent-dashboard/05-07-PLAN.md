<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Human smoke test — verify all six AGT requirements in a live Tauri window</name>
  <what-built>
    A 3-column dashboard with: SessionSidebar (left, 256px) | ChatPane or AgentOutputView (center) | AgentPanel (right, 320px). The AGENTS panel hosts a spawn dialog (model dropdown + task textarea + optional budget), per-agent cards with status badge + truncated task + cumulative cost + expand/stop buttons. Clicking expand replaces the center pane with a read-only view of the agent's conversation. Agents halt automatically when their budget is exceeded, and the OS shows a native notification on Done/Failed.
  </what-built>
  <how-to-verify>
    Launch the app from `tauri-app/`:
    ```
    npm run tauri dev
    ```
    Wait for the window to appear. Then verify each requirement:

    **AGT-01 — Agent roster panel**
    1. Confirm a third (320px) right-side panel is visible labeled `AGENTS`.
    2. Confirm an empty state message "No agents yet. Click + to spawn one." is shown initially.

    **AGT-03 — Spawn dialog**
    3. Click the `+` button in the AGENTS panel header.
    4. Confirm a modal opens with: Model dropdown (populated), Task textarea, Budget (USD, optional) input, Cancel, Spawn Agent buttons.
    5. Enter task: `Smoke test agent A` — leave budget blank — pick any model — click `Spawn Agent`.
    6. Confirm the dialog closes and an agent card appears in the right panel.

    **AGT-05 — Per-agent model**
    7. Open the spawn dialog again. Pick a DIFFERENT model. Task: `Smoke test agent B`. Spawn.
    8. Confirm a second card appears. Note: model name should NOT appear on either card (D-03).

    **AGT-01 visible state — Cards render badge + truncated task + cost**
    9. Confirm each card shows a colored state badge (Idle/Planning/Executing/etc.), the task text, and `$0.0000` cumulative cost. If the agent has progressed, you should see state updates (e.g., Executing with spinner).

    **AGT-02 — Expand for live output**
    10. Click the expand (>) button on agent A's card.
    11. Confirm the center pane swaps from ChatPane to AgentOutputView. The top bar shows the badge + task + cost + X close button. Below is a scrollable area showing any streamed messages (may be empty if ConversationRuntime is still stubbed — that is acceptable for this phase; the point is the swap mechanism works).
    12. Click the X button on the top bar. Confirm the center pane returns to ChatPane.

    **AGT-04 — OS notification**
    13. Spawn a third agent with a very small budget to force a quick Done/Failed: task `Smoke notify`, budget `0.0001`.
    14. Wait for the agent to enter Failed state (budget exceeded should trigger almost immediately if any TurnCompleted fires; if the runtime is still stubbed, manually click Stop to force Failed via the relay).
    15. Confirm a native OS notification appears with title containing the task and body like `Failed — $X.XXXX` or `Done — $X.XXXX`. On Windows in dev, this may appear as a PowerShell-attributed toast — that is expected per RESEARCH.md.

    **AGT-06 — Budget enforcement halts agent**
    16. The agent from step 13 should show the `Failed` red badge and an Error message visible in the expanded view: `Budget exceeded: $X.XXXX`.
    17. Confirm no further state changes occur on that agent — it stays Failed.

    **Layout sanity (D-01, D-02)**
    18. Resize the window to ~1024px width. Confirm all three columns remain visible. Center pane should remain usable (>=400px). No horizontal scrollbar on the window.
  </how-to-verify>

  </resume-signal>
