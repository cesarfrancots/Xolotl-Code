    pub fn set_civ_controller(app_handle: AppHandle, id: String, civ_id: String, controller: Option<String>) -> Result<String, String> {
        let mut snapshot = load_snapshot(&id)?;
        let sanitized = controller
            .map(|s| s.trim().chars().take(64).collect::<String>())

    }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TS frontend -> Tauri IPC (`create_civ_session`, `set_civ_controller`) | User/UI-supplied config (participant names, models, colors) and controller tags cross into the engine and get persisted to disk. |
| Persisted snapshot JSON (`~/.xolotl-code` civilizations dir) -> engine deserialize | Saved files (possibly hand-edited or from older versions) are read back on load. |
| Provider config map (`~/.xolotl-code/config.json`) | Holds API keys (gotcha #2). Models referenced by id only; keys must never enter the snapshot/log/text-state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering / DoS | `create_civ_session` participant list | mitigate | Validate 1-3 participants (Task 1) and reject empty/>3; each model non-empty. Bounds the world size; rejects malformed config. |
| T-01-02 | Spoofing / Injection | `set_civ_controller` controller string | mitigate | Trim + 64-char cap + drop-if-empty before storing (Task 2), so a hostile/overlong tag cannot bloat the snapshot or spoof the leaderboard/text-state row. |
| T-01-03 | Information disclosure | provider API keys leaking into snapshot/log/text-state | mitigate | Phase only reads model ids; never serialize config-map values into civ.model/controller/log. `controller` is a free-form label, not a key. Verified by review — no config-map read in these code paths. |
| T-01-04 | Tampering | malformed/older persisted snapshot deserialize | accept (existing mitigation) | All new fields are `#[serde(default)]`; existing migrate_value_in_place/backfill_snapshot path already hardens load. Low risk: local single-user file the user owns. |
