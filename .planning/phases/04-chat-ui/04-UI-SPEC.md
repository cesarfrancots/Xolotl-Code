## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline code padding, badge padding |
| sm | 8px | Compact element spacing, tool block header padding, message role label gap |
| md | 16px | Default element spacing, chat pane horizontal padding, sidebar item padding |
| lg | 24px | Section padding, message bubble vertical padding |
| xl | 32px | Layout gaps between major zones |
| 2xl | 48px | Top bar height, major section breaks |
| 3xl | 64px | Page-level spacing (unused in Phase 4 — reserved) |

**Exceptions:**
- Chat input textarea minimum height: 64px (4xl = 16px × 4) — matches comfortable single-line entry
- Touch targets for approve/deny/always-allow permission buttons: minimum 36px height (not 44px — desktop-only Tauri app, no touch input requirement)
- Sidebar `w-64` = 256px — intentional non-8pt value; 256px is the natural 4× unit (64px × 4)

## Typography

**Source:** App.css scaffold (Inter font family detected); dark-only system (D-07)

**Scale: 12px / 14px / 16px. Weights: 400 (normal) and 600 (semibold) only.**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.6 | Assistant message text, tool output, sidebar session titles |
| Label | 12px | 400 | 1.4 | Cost footnotes, timestamps, token counts, badge labels, command palette descriptions — distinguished from body by size (12px vs 14px), not weight |
| Heading | 16px | 600 | 1.3 | Top bar model name, session sidebar section header, permission card title |
| Code | 14px | 400 | 1.5 | All `<code>` and `<pre>` blocks — distinguished from body by font family (monospace vs sans) |

**Font family — body:** `Inter, ui-sans-serif, system-ui, -apple-system, Helvetica Neue, Arial, sans-serif`

**Font family — code:** `ui-monospace, 'Cascadia Code', 'Fira Code', Consolas, 'Courier New', monospace`

**Note on prose:** `@tailwindcss/typography` `prose-sm` class governs markdown rendering inside assistant messages. Override `prose` defaults via `@theme` to use the dark palette directly — no `prose-invert` needed (dark-only, D-07). Tailwind v4 `@plugin "@tailwindcss/typography"` syntax is required (not JS config).

## Interaction Design

### Focal Point

On initial load: the input bar placeholder ("Message xolotl...") is the primary visual anchor. During conversation: the last assistant message is the visual anchor.

### Streaming (UI-01)

**rAF buffer pattern** (D-02). Target: 60fps render cadence.

1. Tauri `listen("agent-event:{agentId}")` handler receives `TextDelta` events.
2. Delta string appended to `deltaBuffer` React ref (NOT setState).
3. If no rAF scheduled, schedule one: `requestAnimationFrame(() => { chatStore.appendStreamingContent(buffer); clearBuffer; clearRafId; })`.
4. Zustand action uses functional update: `set(s => ({ streamingContent: s.streamingContent + delta }))` — prevents stale closure.
5. On `TurnCompleted` event: commit `streamingContent` as final message content. Reset `isStreaming`, clear `streamingContent`. Record `TokenUsage` on the message.

**Streaming cursor:** Visible only while `isStreaming === true`. Rendered as a sibling span after the `MarkdownRenderer` output. `animate-pulse` 1s ease-in-out.

### Stop Button (UI-10)

1. User clicks stop.
2. Call `commands.stopAgent(agentId)`.
3. Immediately set `isStreaming = false` in Zustand.
4. Current `streamingContent` is committed to the message list as a partial message with a "(stopped)" label appended in text-muted: `\n\n— *generation stopped*`.
5. The partial message is preserved in the session — not discarded.

### Session Auto-Save (discretion decision)

Auto-save after every `TurnCompleted` event. No manual save required for ongoing sessions. The `/save` slash command is available for explicit named saves. `/load` lists saved sessions. This ensures no data loss from app crashes during a session.

### Tool Block Expand/Collapse (discretion decision)

Default: **collapsed**. Rationale: in a coding-agent context, tool calls are auxiliary information. The user's primary focus is the assistant's response text. Tool details are available on demand. Bash output truncation threshold: **2000 characters** before "show more" link appears.
