# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Parallel Worktrees + Team Orchestration.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Scope: NEW surfaces and extensions only. Existing Phase 4/5 components
> are inherited unchanged unless explicitly listed here as "Extension."

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | detected from `tauri-app/components.json` |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react |
| Font | system-ui (Tailwind default, inherited from Phase 4) |

**Dark-only:** All new components inherit the dark oklch palette established in Phase 4 (D-07).
No light mode. No `dark:` variant classes needed — every component is written dark-first.

## Spacing Scale

Inherited from Phase 4. Multiples of 4 only.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, button gaps |
| md | 16px | Default element spacing, dialog sections |
| lg | 24px | Section padding, dialog body padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

**Phase 6 exceptions:**

| Element | Value | Reason |
|---------|-------|--------|
| Branch label margin-top | 4px (`mt-1`) | Tight visual separation from task text; not a full `sm` gap |
| Group header padding | 8px vertical, 12px horizontal (`py-2 px-3`) | Matches AgentCard `px-3 py-2` base — visual consistency |
| Accordion trigger padding | 8px vertical, 12px horizontal (`py-2 px-3`) | Compact file list; DiffView provides its own inner spacing |
| Accordion content padding | 0px (`p-0`) | DiffView component provides `my-2` + border internally |
| Conflict badge padding | 4px vertical, 8px horizontal (`px-2 py-1`) | Inline badge; sized to multiples of 4 at `text-xs` |
| Merge checkpoint header padding | 16px (`p-4`) | Matches AgentOutputView header padding pattern |
| LaunchTeamDialog role row gap | 8px (`gap-2`) | 4 role rows; tight layout keeps dialog below viewport mid-point |

## Typography

Inherited from Phase 4. No new font sizes introduced.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.4 (`leading-snug`) | Task text in AgentCard, textarea content, dialog body text |
| Label | 12px (`text-xs`) | 400 (normal) | 1.25 | Form labels, state badge text, cost display, header labels |
| Branch label | 12px (`text-xs`) | 400 (normal) | 1.0 (`leading-none`) | Branch name chip — monospace (`font-mono`), single line |
| File path | 12px (`text-xs`) | 400 (normal) | 1.25 | File names in accordion triggers — monospace (`font-mono`) |
| Group header name | 12px (`text-xs`) | 600 (semibold) | 1.25 | Group name in header row — distinguishes it from agent task text |
| Group header status | 12px (`text-xs`) | 400 (normal) | 1.25 | Progress summary "3/4 Done" beside group name |
| Merge checkpoint heading | 14px (`text-sm`) | 600 (semibold) | 1.25 | Branch section headers inside MergeCheckpointView |
| Dialog title | 16px (`text-base`) | 600 (semibold) | 1.25 | LaunchTeamDialog title (follows SpawnAgentDialog pattern) |

**Monospace rule:** All git-derived strings (branch names, file paths) use `font-mono`. All other strings use the default system sans-serif stack.

| Merge button disabled | opacity-50 on primary variant | When any agent in group is non-terminal |

- Pulsing dot element on AgentCard when `state === "Executing"` — outer `animate-ping` ring only (`bg-blue-400/75`), inner dot `bg-blue-500`

**Note on pulsing dot color:** The Executing badge is already `bg-green-900/40 text-green-400`. The pulsing dot uses blue (`bg-blue-400/75`, `bg-blue-500`) to create contrast against the green badge and make the animation visually distinct rather than redundant. This matches the CONTEXT.md specific idea and the Tailwind hero-icon ping pattern.

## Component Specifications

### 1. AgentCard — Extension (D-02, D-03)

**Files:** `tauri-app/src/components/agent/AgentCard.tsx`

**Additions to existing card layout:**

**Branch label** — rendered below the task `<p>` element, above the button row:

```
<code className="text-xs font-mono text-[oklch(0.45_0_0)] mt-1 truncate block max-w-full">
  {agent.branch}
</code>
```

- Color `oklch(0.45_0_0)`: slightly dimmer than `oklch(0.55_0_0)` muted text — visually subordinate to task description
- `truncate` + `block max-w-full`: branch names can be long; must not break card width
- Only rendered when `agent.branch` is a non-empty string (solo agents spawned via SpawnAgentDialog have `branch=""` and show nothing)

**Pulsing dot** — rendered inside the top row alongside `AgentStateBadge`, positioned to the right of the badge:

```
{agent.state === "Executing" && (
  <span className="relative flex h-2 w-2 ml-1">
    <span className="animate-ping absolute inline-flex h-full w-full rounded-full bg-blue-400 opacity-75" />
    <span className="relative inline-flex rounded-full h-2 w-2 bg-blue-500" />
  </span>
)}
```

- `animate-ping` duration: Tailwind default (1s, ease-in-out) — no custom duration
- Size: 8x8px (`h-2 w-2`) — small enough to not compete with the badge
- Placement: `ml-1` after `AgentStateBadge` within the existing `flex items-center` row

**Files:** `tauri-app/src/components/agent/AgentPanel.tsx`

<Button variant="ghost" size="icon" className="h-7 w-7"

- `aria-label`: "Launch team or swarm" (matches title — required for icon-only button accessibility)

**Group header row** — rendered above the AgentCards belonging to each group:

```
<div className="flex items-center justify-between px-3 py-2 mx-2 mt-2 rounded-t-md
     bg-[oklch(0.18_0_0)] border border-neutral-800 border-b-0">
  <div className="flex items-center gap-2 min-w-0">
    <span className="text-xs font-semibold text-[oklch(0.92_0_0)] truncate">{group.name}</span>
    <span className="text-xs text-[oklch(0.55_0_0)] flex-none">{doneCount}/{totalCount} Done</span>
  </div>
  {(group.mergeState === "AllDone" || group.mergeState === "CheckpointOpen") && (
    <Button variant="ghost" size="sm" className="h-6 text-xs px-2 flex-none"
      onClick={() => openMergeCheckpoint(group.id)}>
      View Checkpoint

- `border-b-0` on header + `rounded-t-md`: visually connects header to the agent cards below it
- Agent cards in the group: rendered in a wrapper with `mx-2 border-x border-b border-neutral-800 rounded-b-md` — creates a grouped visual envelope
- Group header name: auto-derived from group data (not specified here — data contract in RESEARCH.md)
- Progress format: exactly `{N}/{Total} Done` — e.g., "3/4 Done", "1/1 Done"
- "View Checkpoint" button: only visible when `mergeState` is `AllDone` or `CheckpointOpen`; hidden when `Pending` or `Merged`
- Ungrouped agents (no `groupId`): render as before — flat list, no group header

### 3. LaunchTeamDialog — New Component (D-04, D-05, D-06)

**Files:** `tauri-app/src/components/agent/LaunchTeamDialog.tsx` (new)

**Pattern:** Follows `SpawnAgentDialog.tsx` exactly — shadcn `Dialog` + form fields.

**Dialog structure:**

```
DialogContent (bg-[oklch(0.16_0_0)] border-neutral-800 text-[oklch(0.92_0_0)])
  DialogHeader
    DialogTitle: "Launch Team"
    DialogDescription: "Configure a role-based team or a parallel swarm."
  [Mode Toggle]
  [Team Form | Swarm Form]
  DialogFooter
    Discard button (ghost)
    Launch button (primary)
```

**Mode Toggle** — rendered below `DialogHeader`, above the form:

```
<div className="flex gap-1 p-1 rounded-md bg-[oklch(0.20_0_0)] w-fit">
  <button className={mode === "team" ? "bg-[oklch(0.28_0_0)] text-[oklch(0.92_0_0)]" : "text-[oklch(0.55_0_0)]"}
    onClick={() => setMode("team")}>Team</button>
  <button className={mode === "swarm" ? "bg-[oklch(0.28_0_0)] text-[oklch(0.92_0_0)]" : "text-[oklch(0.55_0_0)]"}

</div>
```

- Implementation: two plain styled buttons inside a pill container (NOT shadcn Tabs — avoids unnecessary component; NOT ToggleGroup — not installed)
- Active state: `bg-[oklch(0.28_0_0)]` raised surface, text primary
- Inactive state: text muted only
- Pill container: `bg-[oklch(0.20_0_0)]` surface, `rounded-md`, `p-1`, `gap-1`
- Toggle buttons: `rounded-sm px-3 py-1 text-xs font-medium transition-colors`

**Team mode form** — 4 role rows:

<div className="flex flex-col gap-2 py-3 border-b border-neutral-800 last:border-0">

```

Role names (fixed, not user-editable): **Planner**, **Coder**, **Reviewer**, **Tester**

Role task placeholders:
- Planner: "e.g. Break the feature into subtasks and define acceptance criteria"
- Coder: "e.g. Implement the feature based on the plan"
- Reviewer: "e.g. Review the implementation for bugs and code quality"

### 4. MergeCheckpointView — New Component (D-10, D-11, D-12)

**Files:** `tauri-app/src/components/agent/MergeCheckpointView.tsx` (new)

**Mount pattern:** Center pane replacement, identical lifecycle to `AgentOutputView`. Receives `groupId: string` prop. User closes via "Close" button to return to ChatPane.

**Full layout:**

```
<div className="flex flex-col h-full bg-[oklch(0.16_0_0)]">

  {/* Header */}
  <div className="flex items-center justify-between px-4 h-12 flex-none border-b border-neutral-800">
    <span className="text-sm font-semibold text-[oklch(0.92_0_0)]">Merge Checkpoint</span>
    <div className="flex items-center gap-2">
      [merge status indicator]
      <Button variant="ghost" size="icon" className="h-7 w-7"
        title="Close checkpoint"
        aria-label="Close checkpoint"
        onClick={() => openMergeCheckpoint(null)}>
        <X className="h-4 w-4" />
      </Button>
    </div>
  </div>

  {/* Body: per-worktree accordion sections */}
  <ScrollArea className="flex-1">
    <div className="p-4 flex flex-col gap-4">
      {agentsInGroup.map(agent => (
        <WorktreeSection agent={agent} diffs={diffsMap[agent.id]} conflictPaths={conflictPaths} />
      ))}
    </div>
  </ScrollArea>

  {/* Footer */}
  <div className="flex items-center justify-between px-4 py-3 flex-none border-t border-neutral-800">
    {conflictPaths.size > 0 && (
      <span className="text-xs text-yellow-400">
        {conflictPaths.size} conflict{conflictPaths.size !== 1 ? "s" : ""} detected
      </span>
    )}
    {conflictPaths.size === 0 && (
      <span className="text-xs text-[oklch(0.55_0_0)]">No conflicts</span>
    )}
    <Button
      disabled={merging || anyAgentStillRunning}
      title={anyAgentStillRunning ? "All agents must finish before merging" : undefined}
      onClick={() => {
        if (!window.confirm("Merge all worktree branches? This cannot be undone.")) return;
        void handleMerge();
      }}>
      {merging ? "Merging…" : "Approve & Merge"}
    </Button>
  </div>
</div>
```

**Approve & Merge confirmation:** Because the merge is irreversible, clicking "Approve & Merge" must trigger a `window.confirm("Merge all worktree branches? This cannot be undone.")` before invoking `handleMerge`. If the user clicks Cancel in the confirm dialog, the merge does not proceed.

**WorktreeSection** — one accordion per agent branch:

```
<div className="flex flex-col">
  {/* Section heading */}
  <div className="flex items-center gap-2 mb-2">
    <code className="text-xs font-mono text-[oklch(0.92_0_0)]">{agent.branch}</code>
    <span className="text-xs text-[oklch(0.55_0_0)]">·</span>
    <span className="text-xs text-[oklch(0.55_0_0)]">{diffs.length} file{diffs.length !== 1 ? "s" : ""}</span>
  </div>

  {/* File accordion */}
  <Accordion type="multiple" className="flex flex-col gap-1">
    {diffs.map(file => (
      <AccordionItem key={file.path} value={file.path}
        className="bg-[oklch(0.20_0_0)] rounded-md border border-neutral-800">
        <AccordionTrigger
          className="px-3 py-2 text-xs font-mono text-[oklch(0.92_0_0)] hover:bg-[oklch(0.24_0_0)]
                     rounded-md hover:no-underline">
          <span className="flex items-center gap-2 min-w-0">
            <span className="truncate">{file.path}</span>
            {conflictPaths.has(file.path) && (
              <span className="flex-none text-xs bg-yellow-900/40 text-yellow-400 border border-yellow-800
                               px-2 py-1 rounded">
                conflict
              </span>
            )}
          </span>
        </AccordionTrigger>
        <AccordionContent className="px-0 pb-0">
          <DiffView oldStr={file.old_content} newStr={file.new_content} />
        </AccordionContent>
      </AccordionItem>
    ))}
  </Accordion>
</div>
```

**Merge status indicator** (in header, left of close button):

| State | Display |
|-------|---------|
| Loading diffs | `<Loader2 className="h-3 w-3 animate-spin text-[oklch(0.55_0_0)]" />` + `<span className="text-xs text-[oklch(0.55_0_0)]">Loading…</span>` |
| Ready | nothing (clean header) |
| Merging | `<Loader2 className="h-3 w-3 animate-spin text-[oklch(0.55_0_0)]" />` + `<span className="text-xs text-[oklch(0.55_0_0)]">Merging…</span>` |
| Merged | `<span className="text-xs text-emerald-400">Merged</span>` — then auto-close after 1.5s |
| Error | `<span className="text-xs text-red-400">{errorMessage}</span>` |

**Empty state** (agent in group has no changed files):

```
<div className="flex items-center gap-2 px-3 py-4 text-xs text-[oklch(0.55_0_0)] italic">
  No file changes in this worktree.
</div>
```

## State-to-UI Mapping

| State | Visual representation |
|-------|-----------------------|
| Group `mergeState: "Pending"` | Group header shows progress only; no "View Checkpoint" button |
| Group `mergeState: "AllDone"` | "View Checkpoint" button appears in group header; MergeCheckpointView can open |
| Group `mergeState: "CheckpointOpen"` | MergeCheckpointView is the center pane; "View Checkpoint" button still visible for re-opening |
| Group `mergeState: "Merged"` | Group header shows "Merged" in place of "View Checkpoint"; all agent cards in group show Done badge |
| Agent `state: "Executing"` | Pulsing blue dot beside AgentStateBadge; green badge inherited unchanged |
| Agent has `branch` string | `<code>` chip visible below task description |
| Agent has `branch: ""` (solo) | No branch label shown |
| Conflict detected on file | Yellow "conflict" badge inline in AccordionTrigger |
| No conflicts | No badges; "No conflicts" in footer |
| Any agent in group non-terminal | "Approve & Merge" button is `disabled`; tooltip explains why |
| All agents terminal | "Approve & Merge" button is enabled; click triggers confirmation before merge |
| Merge in progress | Button shows "Merging…" and is disabled; header shows spinner |
| Merge complete | Header shows "Merged" in emerald; auto-close after 1.5s |
