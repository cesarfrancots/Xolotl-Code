<objective>
Extend the Rust supervisor to carry per-agent task, model, optional budget, and cumulative cost
state inside AgentHandle. Add a new constructor `AgentHandle::new_with_config` and a new
supervisor method `spawn_agent_with_config(branch, task, model, budget)` that registers a fully
populated handle. Add a `slugify_task` helper (used later by commands.rs) and validation that
rejects non-finite or non-positive budgets. This is the Rust foundation; the Tauri command
layer (plan 03) calls it.

Purpose: Phase 5 requires per-agent model selection (AGT-05), per-agent budget enforcement
(AGT-06), and worktree-from-task-slug spawning (AGT-03). The current AgentHandle has no
field to store any of these, so they vanish after `spawn_agent` returns. Per D-10/D-11 the
budget lives in Rust, not the frontend.

Output: Modified `handle.rs` (new fields + new constructor + accumulate_cost + slugify_task),
`supervisor.rs` (new spawn_agent_with_config method), `mod.rs` (re-export slugify_task).
Existing `AgentHandle::new` and `AgentSupervisor::spawn_agent` remain unchanged for
backwards-compatibility with existing tests.
</objective>

       pub fn slugify_task(task: &str) -> String {

       ```rust
       pub fn spawn_agent_with_config(
           &self,
           branch: &str,
           task: &str,
           model: &str,
           budget_dollars: Option<f64>,
       ) -> Result<AgentId, SupervisorError> {
           // T-5-01: validate budget is positive and finite
           if let Some(b) = budget_dollars {
               if !b.is_finite() || b <= 0.0 {
                   return Err(SupervisorError::InvalidBudget(
                       format!("budget must be > 0 and finite, got {b}")
                   ));
               }
           }

           let agent_id = AgentId::new();
           let worktree_path = self.worktree_manager.add(&agent_id, branch)?;

           let (event_tx, mut event_rx) = mpsc::channel::<AgentEvent>(64);
           let (broadcast_tx, _) = broadcast::channel::<AgentEvent>(64);
           let (cancel_tx, _cancel_rx) = mpsc::channel::<AgentControl>(8);

           let handle = AgentHandle::new_with_config(
               agent_id.clone(),
               worktree_path,
               event_tx,
               broadcast_tx.clone(),
               cancel_tx,
               task.to_string(),
               model.to_string(),
               budget_dollars,
           );

           let broadcast_tx_clone = broadcast_tx.clone();
           tokio::spawn(async move {
               while let Some(event) = event_rx.recv().await {
                   let _ = broadcast_tx_clone.send(event);
               }
           });

           {
               let mut registry = self.registry.lock().unwrap();
               registry.insert(agent_id.clone(), handle);
           }

           Ok(agent_id)
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend→Rust (commands.rs) | Budget value and task string cross from JS into Rust at spawn time. Rust must validate. |
| AgentHandle field access | task/model/budget read in supervisor, commands.rs event relay, and (indirectly) notifications. Slug derivation crosses into `git worktree add` shell call. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 | Tampering | spawn_agent_with_config budget param | mitigate | Reject `!b.is_finite() || b <= 0.0` before registering handle (Task 2). Returns `SupervisorError::InvalidBudget` and never reaches the registry. |
| T-5-02 | Tampering | slugify_task → git worktree add branch arg | mitigate | `slugify_task` (Task 1) strips all non-ASCII-alphanumeric chars to '-', collapses runs, caps slug to 40 chars. No path separators, no shell metacharacters survive. |
| T-5-03 | Spoofing | Notification title content | accept (deferred to plan 06) | Truncation lives in the JS hook layer; backend stores the raw task. Acceptable: WinRT toast renders no HTML. |
</threat_model>
