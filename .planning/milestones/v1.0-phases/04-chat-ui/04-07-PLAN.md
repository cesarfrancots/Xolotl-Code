  <how-to-verify>
Start the app: from `tauri-app/` run `npm run tauri dev`

Wait for the Tauri window to open (may take 30-60s on first build).

**Success Criterion 1 — Streaming echo stub (UI-01):**
> **Note:** `run_agent_turn` uses an echo stub in Phase 4 (per D-03, approved 2026-05-10).
> Real AI streaming requires ConversationRuntime wiring in a follow-on iteration.
> This criterion PASSES when the echo TextDelta streams and finalizes correctly — real AI content is NOT expected.

1. Type any message in the input, e.g. "Hello, is streaming working?"
2. Submit the message
3. Confirm: an echo response appears in the chat thread containing the text "Echo (stub):"
4. Confirm: the text appears progressively via the rAF buffer (not blank screen then sudden text)
5. Confirm: the streaming cursor disappears after the echo completes and the message is committed to the list
6. Confirm: the session cost bar shows $0.0000 — zero tokens is correct for the stub turn

**Success Criterion 2 — Tool call blocks (UI-03, UI-04):**
1. Type a message that triggers a tool: "List all .ts files in the current directory using bash"
2. Confirm: a collapsible tool block appears (collapsed by default) with a Terminal icon
3. Click to expand: confirm bash output is visible, possibly truncated with "Show N more characters"
4. If a file edit tool fires: confirm the DiffView renders with green (+) and red (-) lines

**Success Criterion 3 — Session sidebar + virtualization (UI-05, UI-06):**
1. Send 5+ messages and confirm the session auto-saves (sidebar should show the session after TurnCompleted)
2. Close and reopen the sidebar: confirm the session appears with a timestamp
3. Click the session: confirm it resumes (messages reload)
4. Hover a session row: confirm the trash icon appears; click it: confirm the session is deleted
5. If feasible, check scrolling with 20+ messages loads without jank

**Success Criterion 4 — Model, cost, stop button, permission (UI-07, UI-08, UI-09, UI-10):**
1. Click the model selector in the top bar: confirm the model dropdown lists available models
2. Select a different model: confirm the top bar shows the new model name
3. After a few turns: confirm the session cost bar shows a dollar amount and token count (may be $0.0000 for stub turns)
4. Start a long message and quickly click the Stop button (square icon): confirm streaming halts and "(stopped)" label appears
5. Open DevTools Console, run: `window.__TAURI__.invoke("test_permission_prompt")` — confirm a permission card appears in the chat thread with Allow/Deny/Always Allow buttons; click each and confirm they resolve to the correct badge

**Success Criterion 5 — Slash command palette (UI-11):**
1. Click the input and type `/` — confirm the slash palette popover appears above the input
2. Continue typing `/cl` — confirm the list filters to `/clear`
3. Press Enter (or click) on `/clear` — confirm the session thread clears
4. Type `/help` — confirm an assistant message appears listing all 5 commands
5. Press Escape — confirm the palette closes without executing
  </how-to-verify>
