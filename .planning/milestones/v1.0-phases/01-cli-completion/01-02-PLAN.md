## Trust Boundaries

| Boundary | Description |

| CLI args → budget_limit | User-provided `--budget` value is parsed from untrusted CLI input |
| UsageTracker → footer display | Cost figures come from the runtime's internal state; displayed but not executed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `--budget` arg parsing | mitigate | Parse with `.parse::<f64>()` and return `Err` on invalid input; `NaN` and negative values are accepted but harmless (budget check `cost >= limit` with NaN is always false, effectively no budget) |
| T-02-02 | Information Disclosure | Cost footer exposes session spend | accept | This is the intended feature; cost data is local session state, not a secret |
| T-02-03 | Denial of Service | Malformed `--budget` value crashes CLI | mitigate | `map_err` converts parse error to user-readable `Err` string returned from `parse_args()`; `run()` prints and exits cleanly |
</threat_model>
