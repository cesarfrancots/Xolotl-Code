  ---

  **Checklist B — Kimi K2 Tool-Call Round-Trip (CLI-05)**

  Launch `xolotl --model kimi-coding -Y` (auto-accept tools).

  Send this prompt: `Use the bash tool to run: echo "kimi-tool-call-test"`

  Expected behavior:
  - The model should invoke the `bash` tool (not produce text-only output)
  - The tool runs and returns `kimi-tool-call-test`
  - The model acknowledges the result in its response
  - The cost footer after the turn shows non-zero input/output tokens

  If the model produces text output only without invoking the bash tool, this is a CLI-05 failure.

  **Known edge cases to watch for (from openai.rs research):**
  - If output is garbled or streaming stops mid-response: possible SSE `data:` vs `data: ` prefix issue
  - If the tool call never fires: possible `finish_reason` parsing issue

  ---

  Send this prompt: `Use the bash tool to run: echo "minimax-tool-call-test"`

  Expected behavior: same as Checklist B but for MiniMax.

  **Additional MiniMax edge case to watch for:**
  - Tool call `id` may arrive in a later delta (handled in `openai.rs:781`) — if the tool call gets orphaned (no result returned to model), this fix may have regressed.
