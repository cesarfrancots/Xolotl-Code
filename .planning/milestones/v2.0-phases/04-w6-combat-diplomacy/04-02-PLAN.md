<objective>
Implement deterministic combat: the `civ_strength` seam, `resolve_attack` (bounded casualties on both sides via entity removal, conserved plunder, region flip on a decisive win), the ally no-fight gate, and the queued post-decision combat world pass wired into `advance_civ_turn`. Purpose: Combat is the adversarial, order-sensitive part of Phase 4, so unlike claim/diplomacy/trade (which apply during a civ's own turn in 04-01) attacks are QUEUED during the decision loop and resolved in a single deterministic fixed-order pass after the loop — and BEFORE `resolve_environment`, because the population counter is a mirror re-synced from living axolotl entities at the end of `run_life_cycle`. Casualties therefore remove entities, never decrement the counter. `civ_strength` is the single seam Phase 5's genetic `strength` term plugs into.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM `attack` action → combat resolution | The model's chosen `target` civ id and the attack itself drive entity removal, resource transfer, and territory flips — adversarial mutations of ANOTHER civ's state. |
| Combat RNG → eval replay | Combat outcomes must be byte-identical when a (seed, turn) is replayed (the arena/eval contract). |
| Combat math → world-state integrity | Unbounded casualties/plunder could corrupt the world (instant civ deletion, negative resources). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02 | DoS / availability (game-state) | `resolve_attack` casualties | mitigate | CASUALTY_CAP fraction AND `bound` clamps every loss to strictly less than the side's living count (leave ≥1 survivor) → a single attack can never reduce a defender to 0 (Task 1 test `attack_no_instant_wipeout`). Collapse only through attrition + the existing `should_collapse`. |
| T-04-03 | Repudiation (eval integrity) | combat RNG / resolution order | mitigate | Single `seed^turn^0xC0FF_EE01` rng stream; attacks `sort()`ed to a fixed order; victims selected by sorted entity id; no HashMap iteration / clock / uuid (Tasks 1-2; `resolve_attack_is_deterministic` + `turn_with_combat_is_replay_stable` tests). |
| T-04-04 | Tampering / economic exploit | plunder producing negative resources | mitigate | Plunder `take` is capped at the defender's holdings and drained via `consume` (clamps ≥0); attacker credited the SAME amount → conserved, never negative (Task 1 `plunder_is_bounded_and_conserved`). |
